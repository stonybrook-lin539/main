Relations: Basic notation (Solutions)

Let \(R\) be the relation that connects words to their parts of speech (N for nouns, V for verbs, A for adjectives, P for prepositions, D for determiners, and so on). List the following for English:

  1. export \(R\)
  2. apple \(R\)
  3. \(R\) P
  1. export \(R\) \(= \left \{ N, V \right \}\)
  2. apple \(R\) \(= \left \{ N \right \}\)
  3. \(R\) P is the set of all English prepositions: on, in, to, onto, into, up, down, above, under, next to*, and so on.

For each one of the following, say whether it is a function or just a relation. If you have to make additional assumptions, state them explicitly.

  1. the parent-of relation (e.g. \(j \mathrel{P} m\) for “John is a parent of Mary”)
  2. the parent-of relation in a world where the one-child policy is enforced globally
  3. the relation between a car’s license plate and its owners
  4. the prefix relation, where \(u\) is a prefix of \(v\) iff there is some \(w \in \Sigma^*\) such that \(v = u \cdot w\)
  1. Since an individual can be the parent of more than one individual, this is a relation but not a function.
  2. If we assume that everybody actually obeys this policy, and that the policy has been active for so long that nobody who reached puberty before the policy is still alive, then it is true for all individuals currently alive that each one of them has at most one child. Hence the parent-of relation for individuals currently alive would be a function. (Of course, if we take “in a world” to include the whole history of mankind, then this either includes cases of parents with multiple children from before the policy was enacted, or mankind died out very quickly because each generation’s size would be cut in half.)
  3. If a car can be registered to multiple owners, then this is not a function, but it is still a relation of course.
  4. The prefix relation is not a function. For example, the string ab is a prefix of infinitely many strings, including ab, aba, abab, and so on.

Is the following statement true or false? Justify your answer.

Every relation \(R\) can be regarded as a function that maps \(x\) to \(x \mathrel{R}\).

This statement is true. Suppose, for instance, \(R \mathrel{\mathop:}=\left \{ \left \langle a,b \right \rangle, \left \langle a,c \right \rangle , \left \langle b,c \right \rangle \right \}\). Then we may construct a new function \(r\) from \(R\) such that \(r(a) = \left \{ b,c \right \}\) and \(r(b) = \left \{ c \right \}\), and \(r(c) = \emptyset\). This function \(r\) conveys the same information as \(R\).

Reasoning a bit more abstractly: Let \(r\) be a function such that \(r(x) = x \mathrel{R}\). First, it is indeed the case that \(r\) is a function since, given how we defined \(r\), there can be no two \(y\) and \(z\) such that \(r(x) = y\) and \(r(x) = z\) and \(y \neq z\). If there were two such \(y\) and \(z\), then that would mean that there are two distinct sets that are both identical to \(x \mathrel{R}\), which is impossible. Second, we can construct a relation \(R'\) from \(r\): for every \(x\) in the domain of \(r\), \(R'\) contains every pair \(\left \langle x,y \right \rangle\) such that \(y \in r(x)\). But \(R'\) is exactly the same as \(R\), which shows that we can freely switch between the relation \(R\) and the function \(r\) as we see fit.
